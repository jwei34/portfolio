Improving the Rhode Island Trail Finder: Usability, Accessibility, and Responsiveness

Introduction •  Identifying Usability Problems •  Visual Redesign •  Redesigned Website

Introduction

My goal for this project was to practice the workflow of redesigning a simple website. It involved analyzing and identifying flaws in an existing interface, creating low-fidelity and high-fidelity prototypes for various screen sizes, and building a responsive website based on those prototypes.

The website I picked to redesign is the Rhode Island Trail Finder. I found it by Googling “Rhode Island Trails,” and thought it would be a great website to redesign because a more attractive interface could encourage more people to explore Rhode Island’s various nature offerings.

Building a Conceptual Model

A conceptual model is about how an interface reveals to the user how it works. Websites generally rely on visual cues to communicate their designer-envisioned models to users.

At the top of the interface's hierarchy is the large heading “Rhode Island Trail Finder: Go for a Walk!” effectively communicates that this is a tool for finding trails in Rhode Island (RI). Below this, there is a blue box with interactive input fields, buttons, and dropdown menus for finding and filtering trails. At the bottom, there is a table, presumably with a list of trails that fit the criteria defined in the section above.

My initial impressions were that the interface’s general flow makes sense, and that the visual formats of its input boxes, buttons, and sorting and filtering tools mostly follow conventions that efficiently indicate their functions. However, the placement of many elements feels cluttered and random, without a clear hierarchy, and there is a lot of inconsistency between the different types of search tools.

Identifying Usability Problems

Usability is the capacity of a system to provide a condition for its users to easily perform the tasks while enjoying the experience. It encompasses dimensions such as:

  • Learnability: How easy is it for users to accomplish basic tasks the first time they encounter the design?
  • Efficiency: Once users have learned the design, how quickly can they perform tasks?
  • Memorability: When users return to the design after a period of not using it, how easily can they reestablish proficiency?
  • Errors: How many errors do users make, how severe are these errors, and how easily can they recover from the errors?
  • Satisfaction: How pleasant is it to use the design?

In general, interfaces should be simply, easy to learn, provide appropriate and feedback, and have controls that people expect. Upon closer look at the RI Trail Finder interface, I found a number of usability issues that make it more difficult to navigate than necessary.

Usability Issue #1: Confusing Site Title & Navigation.
  • The cursor changes for all headers upon hover, but some aren't actually clickable. This is poor feedback as the cursor switch is usually associated with clickable buttons.
  • The page title ("Rhode Island Trail Finder: Go for a Walk!") is inconsistent with the menu option (”Trail List/Trail Map”) and the browser tab title (”Explore Rhode Island…”).

Usability Issue #2: Unstandardized Buttons. There are several types of buttons, which makes it hard to learn what function each has.
  • "Go" & "Reset" are very default buttons and work as expected.
  • Pictorial type of trail buttons: At first look, these didn’t even look like buttons. I only realized that they can be used to filter trails after I read the description. Good design shouldn’t require this.
Usability Issue #3: Inconsistent Dropdown Menu Functions.
  • Sort List By & Difficulty are both dropdowns, but one sorts and the other filters. I think these shouldn’t look the same bc their functions are a bit diffrent.
Usability Issue #4: Poor Feedback of "Difficulty" Dropdown Menu.
  • The mechanism of the "Difficulty" dropdown is confusing. Although the ‘Very Easy’ and ‘Easy’ options seem to working correctly, filtering for only trails that have these words in them, the ‘Moderate’ and ‘Difficult’ options look the same, with both include the full list of trails from ‘very easy’ to ‘difficult’. → can’t recognize
Usability Issue #5: Disorganized & misleading use of icons.
  • Serve as buttons at top but just simple icons in the table
  • Disorganized placement in table
  • Sizes aren’t uniform

Accessibility

After identifying usability issues myself, I used WebAIM WAVE to detect possible accessibility problems. Here are 3 more significant problems it identified:

  1. Navigation bar's contrast is too low: AGREE. I think the light yellow text on the semi-transparent dark blue background is quite legible, but the contrast is largely because of the dark banner photo behind it. Without this photo, the dark blue background would be too transparent.
  2. Small text: AGREE. It was easy for me to see this, but I can definitely see it being too small for far-sighted people, e.g. my parents.
  3. No page regions: DISAGREE. WAVE says that the RI Trail Finder has significant page regions that are not defined with appropriate HTML regions such as header, nav, main, footer, as well as other regions such as banner, navigation, main. However, using the Inspect tool reveals that there are, in fact, distinct regions defined in HTML.
